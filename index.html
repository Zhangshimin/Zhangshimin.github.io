数据库的概念

一种存取数据的程序

  • 一种 有很多存取数据的程序

  • 存取

    • 存：能够保持数据避免丢失
    • 取：按照要求取得数据

为什么要使用数据库呢？

  • 较大数据量 数以亿计
  • 网络访问 来自世界各地
  • 并发访问
  • 高性能
  • 事物控制
  • 持久化和数据安全
  • 数据不丢失
  • 查询数据需求逻辑复杂

数据库的类型

  • 关系型数据库
    • 支持sql语句
    • 通过外键进行关联
    • 设计的三范式
      • 字段不可在分
      • 有主键，且非主键字段依赖主键
      • 非主键字段不能相互依赖
  • 非关系型数据库
    • 不支持sql语句
    • 分类:
      • Hadoop 大数据数据库
      • mongoDB 文档型数据库
      • redis 键值数据库
      • Cassandra 架构非常先进的分布式数据库
  • NEWSQL 新型的关系型数据库
    • NewSQL是指这样一类新式的关系型数据库管理系统，针对OLTP（读-写）工作负载，追求提供和NoSQL系统相同扩展性能，且仍然保持ACID和SQL等特性（scalable and ACID and (relational and/or sql -access)）

关于bug的3个概念

软件运行过程中的不同时期有不同的含义

  • Fault 静态过程中产生的，coding
  • Error 在运行过程中，触发了一个Fault而产生的一个中间状态
  • Failure 测试人员或用户观测到的行为

PIE Model

  • Execution 执行通过错误的代码
  • Infection 感染-执行错误代码过程中，触发错误的中间状态
  • Propagation 传播，错误的中间状态必须传播到最后输出，观测到与预期不一致

Discussion

  • Infection：The state of the program must be incorrect
  • Propagation：The infected statc must propagated to cause some output of the program to be incorrect

  • Static Testing & Dynamic Testing
  • Black-Box Testing & While-Box Testing
    Gray-Box Testing： 通过反编译等手段，获取部分代码
  • Testing Level

    • System Testing：build起来以后整个系统
