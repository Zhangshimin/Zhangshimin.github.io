bug的故事

  • Grace Hoper（English）
    - 千年虫
- 第一个bug

L-PIE模型

π模型
来自于 - IEEE Std. 1044,INTRODUCTION TO SOFTWARE TESTING

关于bug的3个概念

软件运行过程中的不同时期有不同的含义

  • Fault 静态过程中产生的，coding
  • Error 在运行过程中，触发了一个Fault而产生的一个中间状态
  • Failure 测试人员或用户观测到的行为

PIE Model

  • Execution 执行通过错误的代码
  • Infection 感染-执行错误代码过程中，触发错误的中间状态
  • Propagation 传播，错误的中间状态必须传播到最后输出，观测到与预期不一致

Discussion

  • Infection：The state of the program must be incorrect
  • Propagation：The infected statc must propagated to cause some output of the program to be incorrect

  • Test Case
    1.Test Input Test Date
2.Test Oracle Expected Output
3.Test Environment
  • Testing & Debugging
  • Validation 确认&Verification
    1.Validation 确认规格文档
    2.Verification 实现是否满足用户需要的规格文档
  • Static Testing & Dynamic Testing
  • Black-Box Testing & While-Box Testing
    Gray-Box Testing： 通过反编译等手段，获取部分代码
  • Testing Level
    • Unit Testing
    • Module Testing
    • Integration Testing：模块级的组合-集合测试
    • System Testing：build起来以后整个系统
  • V Model
  • Testing Process
    • 根据规格获取输入，预期
    • 打包生成测试用例
    • 变成自动化脚本
    • 产生测试结果
